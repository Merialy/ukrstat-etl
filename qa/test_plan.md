# Тест-план: Проєкт «Дані» (Укрстат ETL)
**Версія:** 1.0  
**Дата:** Травень 2026  
**Автор:** QA-інженер (Комолов І.О.)  
---

## 1. Мета тестування

Перевірити коректність перетворення статистичних таблиць Укрстату (Excel/PDF) у структурований формат (JSON, CSV, база даних PostgreSQL) та валідність REST API, що повертає ці дані.
---

## 2. Об'єкт тестування

| Компонент             | Файл/сервіс                      |
|-----------------------|----------------------------------|
| ETL-скрипт            | `scripts/parse_ukrstat.py`       |
| Схема бази даних      | `db/schema.sql`                  |
| REST API              | `api/main.py` (FastAPI)          |
| Вихідні артефакти     | `data/output.json`, `output.csv` |
---

## 3. Scope (що тестуємо)

### У scope:
- Коректність парсингу Excel-файлів Укрстату
- Збереження цілісності даних при конвертації
- Відповідність API специфікації (статус-коди, формат відповіді)
- Обробка відсутніх / нечислових значень (`-`, `...`, `н/д`)
- Пагінація та фільтрація в API

### Поза scope:
- Продуктивність під навантаженням (Performance testing)
- Безпека / автентифікація (не реалізована в прототипі)
- Тестування UI/дашбордів
---

## 4. Типи тестів

| Тип                   | Пріоритет | Інструмент            |
|-----------------------|-----------|-----------------------|

---

## 5. Критерії входу / виходу

**Критерії входу (тести можна запускати):**
- [ ] Excel-файл із тестовими даними доступний
- [ ] Скрипт `parse_ukrstat.py` запускається без помилок
- [ ] API запущений локально (`uvicorn`)

**Критерії виходу (тестування завершено):**
- [ ] Усі критичні тести пройдені
- [ ] Кількість дефектів із пріоритетом P1 = 0
- [ ] Звіт про тестування підготовлено
---

## 6. Тестові сценарії

### TS-01 | ETL: Зчитування Excel

| ID     | Сценарій                                | Очікуваний результат                              |
|--------|-----------------------------------------|---------------------------------------------------|
| TC-001 | Завантажити валідний .xlsx Укрстату     | DataFrame містить N рядків, заголовки розпізнані  |
| TC-002 | Завантажити файл із 3 рядками-шапкою   | Авто-детект заголовку, зайві рядки пропущені      |
| TC-003 | Файл із клітинками `"-"` та `"..."`    | Значення конвертовані у `None`/`NaN`              |
| TC-004 | Файл не існує                           | `FileNotFoundError` із зрозумілим повідомленням   |
| TC-005 | Файл пошкоджений / не є Excel           | Виняток із описом помилки, без краша              |
---

### TS-02 | ETL: Конвертація у JSON

| ID     | Сценарій                                | Очікуваний результат                              |
|--------|-----------------------------------------|---------------------------------------------------|
| TC-010 | Конвертація валідного Excel → JSON      | `output.json` створено, є поля `metadata` і `data`|
| TC-011 | Кількість записів збігається з Excel    | `metadata._total_rows` == кількість рядків у файлі|
| TC-012 | Назви стовпців збережені коректно       | Немає `\n`, зайвих пробілів у ключах JSON         |
| TC-013 | Числові поля є числами, не рядками      | `"2024": 19200`, а не `"2024": "19200"`           |
| TC-014 | Файл валідний JSON                      | `json.loads()` не кидає виняток                   |
---

### TS-03 | ETL: Конвертація у CSV

| ID     | Сценарій                                | Очікуваний результат                              |
|--------|-----------------------------------------|---------------------------------------------------|
| TC-020 | Конвертація валідного Excel → CSV       | `output.csv` створено із заголовками              |
| TC-021 | Кількість рядків збігається             | Рядків у CSV == рядків у Excel (не рахуючи шапку) |
| TC-022 | Кодування utf-8-sig                     | Файл відкривається в Excel без кракозябр          |
| TC-023 | Відсутні значення представлені як порожні клітинки | Не `"None"` або `"NaN"` у CSV           |
---

### TS-04 | API: Загальні перевірки

| ID     | Сценарій                                | Очікуваний результат                              |
|--------|-----------------------------------------|---------------------------------------------------|
| TC-030 | GET /health                             | `200 OK`, `{"status": "ok"}`                      |
| TC-031 | GET /api/v1/stats                       | `200 OK`, є поля `total`, `page`, `data`          |
| TC-032 | GET /api/v1/stats/regions               | `200 OK`, список рядків у `regions`               |
| TC-033 | GET /api/v1/stats/summary?year=2024     | `200 OK`, є поля `min`, `max`, `average`          |
| TC-034 | GET /api/v1/stats/region/Київ           | `200 OK`, є поле `timeseries` із 5 точками        |
---

### TS-05 | API: Негативні сценарії

| ID     | Сценарій                                | Очікуваний результат                              |

## 7. Дефекти та пріоритети

| Пріоритет | Опис                                              |
|-----------|---------------------------------------------------|
| P1        | Втрата даних при конвертації, краш скрипту        |
| P2        | Неправильні типи даних, некоректні статус-коди    |
| P3        | Косметичні проблеми, помилки у повідомленнях      |
